What it does

Built to actually work,
not to impress a demo.

Every feature is either working or marked clearly as planned. No vaporware.

🧠
On-Device Inference
LlamaCpp JNI bridge runs quantized models directly on your phone's CPU. No API calls. No connectivity needed. Works on the subway.
llama.cpp · JNI · C++
🔒
Encrypted Storage
All conversations stored in SQLCipher-encrypted database. Biometric gate on app open. Your data is not readable without your fingerprint.
SQLCipher · BiometricGate
🌡
Thermal Awareness
TawsGovernor monitors device temperature and automatically steps down model tier when the phone gets hot. Your battery thanks you.
ThermalMonitor · TawsGovernor
🔗
Desktop Offload
Tailscale mesh networking lets your phone offload heavy inference to your PC over your local network. Control your PC from your phone, see results on mobile.
Tailscale · MCP · Python server
🤖
ReAct Agent
Thought → Action → Observation loop. Agent capabilities include app launching, web research, file system access, clipboard, datetime, and calculator.
ReActEngine · ToolRegistry
💾
Long-Term Memory
HindsightMemory compresses conversations into long-term context. Mias remembers what you've told it across sessions — without sending it to a server.
HindsightMemory · EmbeddingEngine
🎙
Voice + STT/TTS
Speech-to-text and text-to-speech engines built in. Voice is the primary interaction mode — the chat screen is secondary.
SpeechEngine · TtsEngine
🔄
Self-Evolution
Background WorkManager job runs every 6 hours to consolidate conversation knowledge, update personality parameters, and optimize for your usage patterns.
EvolutionWorker · KnowledgeConsolidator
📡
Model Hub
Browse and download models directly from HuggingFace. The InferenceOrchestrator routes queries to the right model based on task type and device state.
HuggingFaceRegistry · ModelManager

Contributing

Two types of
valid contributions.

Contributions need to be focused. Read the full CONTRIBUTING.md before opening a PR.

01 / BUG FIX
Fix something broken
Found a bug? Reproduce it, fix it, commit it. One commit per bug with a clear message describing what broke and what you changed.
fix: ReActEngine infinite loop on unknown tool
— added ToolRegistry validation guard
02 / FEATURE IMPROVEMENT
Make one thing better
Pick a single feature and improve it meaningfully. Open an issue first. One PR per feature area. No half-finished work.
improvement: VoiceChatScreen — added
tap-to-toggle vs hold-to-speak mode selection
